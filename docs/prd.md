# 需求文档

## 1. 应用概述

**应用名称**：匿名互评系统

**应用描述**：团队成员之间互相填写评价的匿名互评网页应用，支持管理员创建评价周期、成员匿名填写评价、查看个人收到的评价及统计数据。

## 2. 用户与使用场景

**目标用户**：
- 管理员：负责系统管理、用户管理、评价周期管理
- 填写+查看+被评价者：可填写对他人的评价，可查看自己收到的评价
- 查看+被评价者：只能查看自己收到的评价

**核心使用场景**：
- 管理员创建评价周期，团队成员在周期内匿名填写对其他成员的评价
- 成员查看自己收到的匿名评价及统计数据
- 管理员查看全局评价数据和统计

## 3. 页面结构与功能说明

### 页面层级结构

```
匿名互评系统
├── 登录页
├── 管理后台（管理员）
│   ├── 用户管理
│   ├── 评价周期管理
│   └── 全局评价统计
├── 评价填写页（填写+查看+被评价者）
├── 个人评价查看页（所有角色）
└── 评价汇总统计页（所有角色）
```

### 3.1 登录页

**功能说明**：
- 用户输入账号密码登录
- 登录成功后根据角色权限跳转到对应页面
- 使用Supabase进行登录认证

### 3.2 管理后台

**访问权限**：仅管理员可访问

#### 3.2.1 用户管理

**功能说明**：
- 查看所有用户列表，包含用户名、角色信息
- 新增用户，设置用户名、密码、角色（管理员/填写+查看+被评价者/查看+被评价者）
- 编辑用户信息，修改角色权限
- 删除用户

#### 3.2.2 评价周期管理

**功能说明**：
- 查看所有评价周期列表，包含周期名称、开始时间、结束时间、状态
- 创建新评价周期，设置周期名称、开始时间、结束时间
- 编辑评价周期信息
- 删除评价周期
- 关闭或开启评价周期

#### 3.2.3 全局评价统计

**功能说明**：
- 选择评价周期，查看该周期内所有成员的评价数据
- 展示每个成员在5个维度的评分统计（数据质量、个人效率、服从工作安排、工作积极性、其他帮助）
- 查看每个成员收到的所有评价详情（包含评价人身份、评价内容、备注）

### 3.3 评价填写页

**访问权限**：仅「填写+查看+被评价者」角色可访问

**功能说明**：
- 选择当前开启的评价周期
- 展示所有其他成员列表（不包含自己）
- 对每个成员填写5个维度的评价：
  - 数据质量：勾选评价+填写备注
  - 个人效率：勾选评价+填写备注
  - 服从工作安排：勾选评价+填写备注
  - 工作积极性：勾选评价+填写备注
  - 其他帮助：勾选评价+填写备注
- 提交评价，保存到后端
- 评价提交后，填写人身份对被评价人不可见

### 3.4 个人评价查看页

**访问权限**：所有角色可访问

**功能说明**：
- 选择评价周期
- 查看自己在该周期内收到的所有评价
- 评价以匿名方式展示，不显示填写人身份
- 展示每条评价的5个维度内容（评价+备注）

### 3.5 评价汇总统计页

**访问权限**：所有角色可访问

**功能说明**：
- 选择评价周期
- 展示自己在该周期内5个维度的评分统计
- 统计维度包括：数据质量、个人效率、服从工作安排、工作积极性、其他帮助
- 展示每个维度的评价分布情况

## 4. 业务规则与逻辑

### 4.1 匿名机制

- 填写人提交评价后，被评价人查看评价时不显示填写人身份
- 管理员查看全局评价统计时可看到填写人身份

### 4.2 角色权限

- 管理员：可访问管理后台所有功能、个人评价查看页、评价汇总统计页
- 填写+查看+被评价者：可访问评价填写页、个人评价查看页、评价汇总统计页
- 查看+被评价者：仅可访问个人评价查看页、评价汇总统计页

### 4.3 评价周期规则

- 同一时间可存在多个评价周期，但仅一个周期处于开启状态
- 评价周期开启时，有填写权限的用户可填写评价
- 评价周期关闭后，用户不可再填写评价，但可查看已提交的评价

### 4.4 评价填写规则

- 用户不可对自己填写评价
- 每个维度必须勾选评价，备注为可选项
- 同一评价周期内，用户对同一成员只能提交一次评价

### 4.5 数据存储

- 使用Supabase存储用户数据、评价周期数据、评价数据
- 评价数据需保存填写人ID、被评价人ID、评价周期ID、5个维度的评价内容和备注

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 用户登录失败 | 提示账号或密码错误 |
| 无评价周期开启时访问评价填写页 | 提示当前无可填写的评价周期 |
| 用户尝试对自己填写评价 | 系统自动过滤，不展示自己 |
| 用户重复提交同一成员的评价 | 提示已提交过评价，不可重复提交 |
| 管理员删除用户 | 该用户的历史评价数据保留 |
| 管理员删除评价周期 | 该周期内的所有评价数据删除 |
| 评价周期结束后用户尝试填写评价 | 提示评价周期已结束，不可填写 |

## 6. 验收标准

1. 管理员登录系统，创建评价周期（设置周期名称、开始时间、结束时间）
2. 管理员新增用户，设置用户角色为「填写+查看+被评价者」
3. 用户登录系统，进入评价填写页，选择评价周期
4. 用户对其他成员填写5个维度的评价（勾选评价+填写备注），提交评价
5. 用户进入个人评价查看页，查看自己收到的匿名评价
6. 管理员进入全局评价统计页，查看所有成员的评价数据和统计
